快速開始

日常啟動只要點桌面捷徑，等畫面出來就能用。

啟動 App

點桌面捷徑 Leo work.lnk（或執行 start-leo-work.bat）。它會自動清掉舊連線、起服務，幾秒後開瀏覽器。

開啟畫面

瀏覽器進入 http://localhost:1420。畫面會等背景服務就緒後才顯示，這是正常的。

開始用

頂部有兩個分頁：工作區（錄音 / 逐字稿 / 分析 / 匯出）與 🦉 記憶聊天（跨會議問答）。

💡 需要網路與金鑰語音轉錄、即時逐字稿、AI 助理對話走 Google Gemini（有免費額度），需要設定 GEMINI_API_KEY 並連網。加密、向量記憶等本地功能離線也能跑。

核心功能

一場會議從「錄」到「整理成文件」需要的，都在裡面。

🎙️

錄音 + 混合式即時逐字稿

按下錄音邊講邊出字，停止後再整檔精修成乾淨版。

  • 錄音中：即時跳出「🔴 即時粗稿」（速度優先，可能有小瑕疵）
  • 按停止：自動用乾淨、帶 [mm:ss] 發言人: 的精修版覆蓋
  • 超過 15 分鐘會自動續錄、不中斷
🌐

轉錄語言可選

逐字稿面板右上下拉，控制停止後精修版的語言。

  • 自動（預設）：用原語言；非中文句後面附全形括號繁中翻譯
  • 一律繁中：全部轉繁體中文（最穩）
  • 一律英文：全部轉英文
🦉

AI 助理（聊天＋匯出一體）

工作區底部的主互動面板，預設展開、可放大。

  • 對話式問「這份逐字稿」的問題，並結合跨會議記憶回答
  • 記得多輪對話脈絡
  • 面板上方就有匯出列：📋複製 / ⬇.md / 📄Word / 📊Excel / 📽PPT
📊

主動式分析

按「分析」一鍵產出結構化結果。

  • 主題、摘要、⚠️ 衝突提醒、行動方針表（任務 / 負責人 / 截止日）
  • 會橫向比對你過去的會議，找出前後矛盾
  • 分析＋行動方針併成 1 次請求，省免費額度
🗂️

跨會議全局記憶

每場會議自動切片進向量庫，之後可跨場檢索。

  • 本地 LanceDB 向量庫、離線嵌入模型
  • 問答時自動把相關的舊會議內容找回來當依據

翻譯

📤

.md / Word / Excel / PPT，瀏覽器端離線產檔、不上傳。

  • AI 客製：跟 AI 討論完要怎麼整理，再產出

雙源收音

除了電腦麥克風，還能用電腦系統聲或手機當麥克風。

  • 🖥️ 電腦系統混音、📱 手機掃 QR 收音
  • VU 訊號條即時顯示音量

完整使用流程（一場會議從頭到尾）

最常見的用法：用電腦麥克風錄一場會、整理成文件。

進「工作區」分頁

確認逐字稿面板在畫面中央。

按 🎙 錄音

開始計時並收音；錄音中會即時跳出「🔴 即時粗稿」讓你看到進度。

（可選）先選好轉錄語言

右上下拉選「自動 / 一律繁中 / 一律英文」。會議若以中文為主，選「一律繁中」最穩。

按停止

系統自動把整段送精修，用乾淨版（繁體、時間戳、發言人）覆蓋粗稿，並落地保存。

按「分析」

產出摘要、衝突提醒、行動方針表，並比對歷史會議。

用 🦉 AI 助理追問 / 討論

例如「幫我把結論濃縮成三點」「這場跟上次的決議有沒有衝突」。

匯出

在 AI 助理上方匯出列選格式（Word / Excel / PPT / .md），直接下載。

兩套收音系統的差別（重要）

App 有兩套獨立的收音入口，搞混會雙錄又撞限流。請每次只挑一套。

🎙 下方逐字稿面板

來源：電腦本機麥克風
  • 按 🎙 錄音 → 邊講邊出即時稿
  • 按停止 → 自動精修並填入會議逐字稿
  • 最單純的一般會議用這個

🖥️📱 上方收音列（頂部 header）

來源：電腦系統聲 或 手機
  • 選 🖥️ 電腦系統 / 📱 手機收音（掃 QR）
  • 收音時上方顯示即時粗稿（預覽）
  • 按停止後要手動點「✨ 精修並帶入會議」才會填入
⚠️ 操作重點 上方收音列的即時稿只是預覽，永遠不會自動流入會議稿——一定要「停止 → 點 ✨ 精修並帶入會議」。 若停在「手機收音中」沒按停止，就不會有東西進去。別同時開兩套。
📱 手機收音怎麼連 點「📱 手機收音」後畫面出現 QR；手機掃描開啟網頁（自簽憑證，首次要點「信任 / 繼續」），允許麥克風後就開始把聲音傳回電腦。需同一區網。

線上會議怎麼錄（VooV / Zoom / Teams）

Leo work 不接會議軟體的 API、也不以機器人身分入會——它在作業系統層收音，所以 VooV、Zoom、Teams 對它來說完全一樣，方法通用。

✅ 正確做法：上方收音列選「🖥️ 電腦系統」 這個來源會用 FFmpeg 同時抓 你的麥克風（你這端）＋ 系統 loopback（喇叭輸出＝會議對方的聲音），混成一軌——正是為線上會議設計的。下方 🎙 錄音只收麥克風、收不到對方，不適合線上會議。

前提（關鍵）：要先有 loopback 裝置

程式只認得名稱含這些字的裝置：立體聲混音 / Stereo Mix / CABLE Output / virtual-audio / VoiceMeeter / What U Hear。沒有的話會降級成「僅錄麥克風」，對方的聲音就收不到。

  • 最簡單：開 Windows 內建「立體聲混音」——右下角喇叭圖示右鍵 → 音效設定 → 更多音效設定 → 錄製分頁 → 空白處右鍵「顯示已停用的裝置」→ 對立體聲混音右鍵啟用。
  • 沒有立體聲混音（部分筆電 / USB 耳麥沒有）：裝虛擬音源 VB-Audio Virtual Cable 或 VoiceMeeter。

操作步驟

先正常進 VooV / Zoom / Teams 開會。

Leo work 頂部收音列點 🖥️ 電腦系統。

看 VU 訊號條：你講話、對方講話兩邊都要有跳動（只有一邊跳＝loopback 沒設好）。

散會按停止 → 點「✨ 精修並帶入會議」→ 語言選一律繁中 → 填入會議逐字稿。

按「分析」→ 用 🦉 AI 助理整理 → 匯出 Word / PPT。

耳機 vs 外放

情況結果
有 loopback（立體聲混音 / 虛擬線）✅ 戴耳機也收得到對方（它抓的是播放串流，不是環境音）——首選
沒 loopback ＋ 外放⚠️ 麥克風會收到喇叭傳出的對方聲音，但品質差、有回音 / 雜音
沒 loopback ＋ 戴耳機❌ 對方聲音進耳機、麥克風收不到 → 只錄到你自己
⏱️ 單次精修上限約 60 分鐘 「✨ 精修並帶入會議」的緩衝上限為 60 分鐘（超過會提醒只精修前段）。大檔會自動改用 Gemini Files API 上傳，所以一小時內的會議都能完整精修；但按下精修後「上傳＋轉錄」需要等一段時間（接近一小時的會議可能要等 1～2 分鐘，屬正常）。超過 1 小時的會議建議中途分段：停止 → ✨精修帶入 → 再重新收音。
🔔 開錄前先告知與會者 錄製會議涉及他人聲音，依場合與規範，開始前請先讓與會者知道並取得同意。

匯出會議記錄

在 🦉 AI 助理面板上方的匯出列操作（需先按「分析」有結果）。

方式怎麼觸發產出
預設範本
零 API、最快		 沒跟 AI 討論，直接點格式鈕 本機把分析結果排版：Word＝完整記錄含逐字稿；Excel＝概要＋各表分頁；PPT＝封面＋每節一張
AI 客製
討論完再產出		 先在助理裡多輪討論要怎麼整理，再點格式鈕 AI 依「討論脈絡＋格式＋會議資料」重組。例：「PPT 只放結論和數字」「Word 用公文語氣加風險建議」「Excel 行動方針多一欄優先級」
📈 想要圖表 → 用 PPT 只要 AI 把數字整理成表格（第一欄項目＋其餘欄純數字），PPT 會自動補一張圖表投影片（單欄→圓餅、多欄→長條）。Word／Excel 則以資料表呈現。
🛟 韌性 AI 重組偶爾會因為過載或安全過濾失敗，這時會自動退回預設範本匯出，至少一定產得出檔，不會卡住（可再試一次）。

整頁「記憶聊天」

頂部分頁切到「🦉 記憶聊天」——不綁當前會議，純粹跨所有會議的記憶問答。

✨

空白時

顯示歡迎畫面：大輸入框 ＋ 一排建議卡，點一下就能起頭問。

💬

有對話時

變成訊息串 ＋ 底部輸入列，可隨時「＋ 新對話」回到首頁。

和工作區底部的 AI 助理走同一套問答，差別是這裡不帶當前逐字稿，只靠跨會議向量記憶。 適合問「我之前哪幾場提過 X」「上季的決議是什麼」這類橫跨多場會議的問題。 （部分建議卡是範例提示詞，若記憶庫沒相關內容，AI 會誠實回查無。）

注意事項 / 已知行為

現象說明
即時粗稿出現簡體 / 別國語言 / 字間有空格這是 STT 原始輸出的已知行為，即時稿只是預覽。以停止後的精修版為準（精修版繁體、有標點與發言人）。
AI 提示「請稍候 N 秒再試」Gemini 免費額度每分鐘有上限，等幾秒～1 分鐘即可恢復；系統會自動短等待重試。連點會一直撞同一個窗口。
提示「每日額度用完」當天免費額度用光，隔日恢復；或在 .env 換成另一個 Google 帳號的金鑰（免費額度是按帳號算）。
手機掃 QR 連不上需與電腦同一區網；首次要在手機點「信任憑證 / 繼續前往」（自簽 HTTPS 屬正常）。
電腦系統聲錄不到Windows 需開「立體聲混音」或裝虛擬音源；偵測不到時會自動退化為只錄麥克風。
🔵 藍牙同步介面有保留，但目前未啟用（需特定錄音卡硬體），暫不可用。

設定（給需要動到的人）

設定都在專案根目錄的 .env，改完要重啟服務（重新點捷徑）才生效。

變數用途
GEMINI_API_KEY轉錄 / 即時逐字稿 / AI 助理對話必填（Google AI Studio 申請，有免費額度）
GEMINI_MODEL對話 / 分析 / 整檔轉錄共用的模型（目前 gemini-3.5-flash）
GEMINI_LIVE_MODEL即時逐字稿用的 Live 模型（gemini-3.1-flash-live-preview）
LLM_PROVIDER分析/翻譯用哪家：gemini（現用）/ ollama（本地免費離線）/ claude（付費）
ENCRYPTION_SALT加密金鑰推導用。別亂改——改了舊的加密會議檔會無法解密
🔐 安全 .env、加密會議檔、資料庫都已排除在版控外，金鑰不會進 git。所有錄音 / 逐字稿以 AES-256-GCM 加密儲存，被竄改會直接拒絕解密，不會回傳亂碼。
